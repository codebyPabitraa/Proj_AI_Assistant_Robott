# Module 06 — Eye Animations & Expressions

**Role:** Display emotional expressions on OLED screens to make robot feel alive.
**Hardware:** Raspberry Pi 4, OLED Display (1.3" SSD1306, I2C), or TFT LCD
**Dependencies:** luma.oled / Adafruit SSD1306, PIL (Pillow), pygame (optional)

---

## Tech Stack

| Component | Library | Why |
|---|---|---|
| OLED Driver | luma.oled (SSD1306) | Pi-compatible I2C OLED library |
| Image Rendering | Pillow (PIL) | Draw shapes and text on frames |
| Animation | Frame sequence loop | Simple, no GPU needed |
| State Manager | Python enum | Track robot state → map to expression |

---

## Architecture

```
[Robot State: IDLE / LISTENING / THINKING / SPEAKING / GREETING]
  → state_manager.py        (track current robot state)
  → expression_mapper.py    (map state → expression name)
  → eye_renderer.py         (draw eye frame using PIL)
  → oled_display.py         (push frame to OLED via I2C)
```

## Expression States

| State | Eye Look | When |
|---|---|---|
| IDLE | Slow blink, relaxed oval | Default / waiting |
| LISTENING | Wide open, attentive | After wake word |
| THINKING | Looking up + squint | LLaMA processing |
| SPEAKING | Animated with mouth sync | TTS playing |
| GREETING | Happy curved eyes | Known face detected |
| SURPRISED | Very wide circle eyes | Unexpected object detected |
| SLEEPING | Closed line eyes | Low activity / standby |

## Code

### oled_display.py
```python
from luma.core.interface.serial import i2c
from luma.oled.device import ssd1306
from PIL import Image

# I2C address is usually 0x3C for SSD1306
serial = i2c(port=1, address=0x3C)
device = ssd1306(serial, width=128, height=64)

def show_image(pil_image: Image.Image):
    """Push a PIL image to the OLED display."""
    # Convert to 1-bit mode for OLED
    image = pil_image.convert("1")
    device.display(image)

def clear_display():
    """Clear the OLED."""
    device.clear()
```

### eye_renderer.py
```python
from PIL import Image, ImageDraw
import math

WIDTH, HEIGHT = 128, 64
CENTER_X, CENTER_Y = WIDTH // 2, HEIGHT // 2

def draw_idle_eyes(frame: int = 0) -> Image.Image:
    """
    Slow blinking oval eyes.
    frame: animation frame number (for blink timing)
    """
    img = Image.new("1", (WIDTH, HEIGHT), 0)
    draw = ImageDraw.Draw(img)

    # Blink every 60 frames (simulate ~2s at 30fps)
    blink = (frame % 60) < 5

    if blink:
        # Closed eyes — horizontal lines
        draw.line([(35, 32), (55, 32)], fill=1, width=2)
        draw.line([(73, 32), (93, 32)], fill=1, width=2)
    else:
        # Open oval eyes
        draw.ellipse([(30, 22), (60, 42)], outline=1, width=2)
        draw.ellipse([(68, 22), (98, 42)], outline=1, width=2)
        # Pupils
        draw.ellipse([(42, 28), (50, 36)], fill=1)
        draw.ellipse([(80, 28), (88, 36)], fill=1)

    return img

def draw_listening_eyes() -> Image.Image:
    """Wide open attentive eyes."""
    img = Image.new("1", (WIDTH, HEIGHT), 0)
    draw = ImageDraw.Draw(img)

    draw.ellipse([(26, 16), (62, 48)], outline=1, width=2)
    draw.ellipse([(66, 16), (102, 48)], outline=1, width=2)
    # Large pupils
    draw.ellipse([(39, 26), (51, 38)], fill=1)
    draw.ellipse([(77, 26), (89, 38)], fill=1)

    return img

def draw_thinking_eyes() -> Image.Image:
    """Eyes looking up + slight squint."""
    img = Image.new("1", (WIDTH, HEIGHT), 0)
    draw = ImageDraw.Draw(img)

    # Squinted (flatter ellipse)
    draw.ellipse([(30, 26), (60, 40)], outline=1, width=2)
    draw.ellipse([(68, 26), (98, 40)], outline=1, width=2)
    # Pupils moved up
    draw.ellipse([(41, 26), (49, 32)], fill=1)
    draw.ellipse([(79, 26), (87, 32)], fill=1)

    return img

def draw_greeting_eyes() -> Image.Image:
    """Happy curved ^ shape eyes (anime happy)."""
    img = Image.new("1", (WIDTH, HEIGHT), 0)
    draw = ImageDraw.Draw(img)

    # Draw arc for happy eyes
    # Left eye arc (^)
    draw.arc([(28, 20), (60, 50)], start=200, end=340, fill=1, width=3)
    # Right eye arc (^)
    draw.arc([(68, 20), (100, 50)], start=200, end=340, fill=1, width=3)

    return img

def draw_sleeping_eyes() -> Image.Image:
    """Closed eyes — thin horizontal lines."""
    img = Image.new("1", (WIDTH, HEIGHT), 0)
    draw = ImageDraw.Draw(img)

    draw.line([(30, 32), (60, 32)], fill=1, width=3)
    draw.line([(68, 32), (98, 32)], fill=1, width=3)

    # Zzz text
    draw.text((100, 10), "z", fill=1)
    draw.text((108, 4), "Z", fill=1)

    return img

def draw_surprised_eyes() -> Image.Image:
    """Large circle eyes."""
    img = Image.new("1", (WIDTH, HEIGHT), 0)
    draw = ImageDraw.Draw(img)

    draw.ellipse([(22, 12), (62, 52)], outline=1, width=2)
    draw.ellipse([(66, 12), (106, 52)], outline=1, width=2)
    # Small pupils
    draw.ellipse([(38, 28), (46, 36)], fill=1)
    draw.ellipse([(82, 28), (90, 36)], fill=1)

    return img

EXPRESSION_MAP = {
    "IDLE":      draw_idle_eyes,
    "LISTENING": draw_listening_eyes,
    "THINKING":  draw_thinking_eyes,
    "GREETING":  draw_greeting_eyes,
    "SLEEPING":  draw_sleeping_eyes,
    "SURPRISED": draw_surprised_eyes,
}

def get_eye_frame(expression: str, frame: int = 0) -> Image.Image:
    """Returns PIL image for given expression."""
    func = EXPRESSION_MAP.get(expression, draw_idle_eyes)
    try:
        return func(frame)  # pass frame for animated expressions
    except TypeError:
        return func()       # non-animated expressions don't take frame arg
```

### state_manager.py
```python
from enum import Enum
import threading

class RobotState(Enum):
    IDLE      = "IDLE"
    LISTENING = "LISTENING"
    THINKING  = "THINKING"
    SPEAKING  = "SPEAKING"
    GREETING  = "GREETING"
    SURPRISED = "SURPRISED"
    SLEEPING  = "SLEEPING"

_current_state = RobotState.IDLE
_lock = threading.Lock()

def set_state(state: RobotState):
    global _current_state
    with _lock:
        _current_state = state
        print(f"[Eyes] State → {state.value}")

def get_state() -> RobotState:
    with _lock:
        return _current_state
```

### eye_loop.py (runs in background thread)
```python
import time
import threading
from modules.animations.state_manager import get_state, RobotState
from modules.animations.eye_renderer import get_eye_frame
from modules.animations.oled_display import show_image

FRAME_RATE = 15  # FPS

def eye_animation_loop():
    """Continuously updates OLED with correct eye expression."""
    frame = 0
    while True:
        state = get_state()
        img = get_eye_frame(state.value, frame)
        show_image(img)
        frame = (frame + 1) % 1000
        time.sleep(1 / FRAME_RATE)

def start_eye_thread():
    """Start eye animation in background thread."""
    t = threading.Thread(target=eye_animation_loop, daemon=True)
    t.start()
    print("[Eyes] Animation thread started.")
```

## Hardware Integration

```
SSD1306 OLED (I2C) Wiring:
  VCC  → 3.3V (Pi pin 1)
  GND  → GND  (Pi pin 6)
  SDA  → GPIO 2 (Pi pin 3) — I2C Data
  SCL  → GPIO 3 (Pi pin 5) — I2C Clock

For two OLED screens (left eye + right eye):
  - Use I2C address 0x3C for one, 0x3D for the other
  - OR use two separate I2C buses (Pi has I2C0 and I2C1)
  - OR use SPI OLEDs for easier multi-display setup
```

## ⚠️ Difficulties

| Issue | Cause | Fix |
|---|---|---|
| OLED not detected on I2C | I2C not enabled | Run `raspi-config` → enable I2C; check with `i2cdetect -y 1` |
| luma.oled install fails | Missing dependencies | `sudo apt install python3-dev libfreetype6-dev libjpeg-dev` first |
| Eye animation too slow | Thread sleeping too long | Reduce `time.sleep` — but don't go below 1/30 on Pi |
| Two OLEDs showing same image | Same I2C address | Solder address jumper on one OLED to change to 0x3D |
| PIL drawing not showing on OLED | Wrong image mode | Always convert to `"1"` (1-bit) before display |
| Eye state not updating during LLaMA | Blocking main thread | State manager is thread-safe; eye loop runs independently ✓ |

## Integration Points

- **Module 01 (Voice):** `LISTENING` on wake word → `THINKING` during LLaMA → `SPEAKING` during TTS
- **Module 03 (Vision):** `GREETING` when known face detected; `SURPRISED` when unknown person
- **Module 05 (Sound):** Animate eyes to look left/right based on sound direction
- **Module 07 (App):** Stream current eye expression state to web app for monitoring

## Scalability Notes (Production)

- Replace OLED with **round TFT LCD (GC9A01)** — supports full color, looks like real eyes
- Use **two separate displays** — left eye + right eye for more expressive look
- Add **lip sync animation** — analyze TTS audio amplitude → animate a mouth bar under eyes
- Pre-render all animation frames as PNG sequences → smoother playback
- On Jetson production: use **HDMI display** with full OpenGL animated face (like Boston Dynamics Spot)
